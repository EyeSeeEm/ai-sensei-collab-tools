Summary

ü§ñ OpenClaw on VPS

Self-hosted bot on a dedicated server. Accessible via Telegram, Discord, WhatsApp, Slack, and Email. Can have its own email address (e.g., sensei@aisensei.com) so anyone can just write to it.

Shared context - everyone talks to the same bot with the same knowledge base. Full customization possible: system prompts, MCP tools, web access.

Best for
Team collaboration where everyone needs access via their preferred chat app. Marketing content, strategy discussions, shared AI assistant.

üß† Claude.ai Projects

Web-based projects with persistent knowledge files. Upload documents, set custom instructions. Zero infrastructure to maintain.

Limitation: Each person has separate conversations. Can't see each other's chats in real-time. Web-only access.

Best for
Individual work with shared reference materials. Good fallback if VPS maintenance becomes too much.

üí¨ Claude for Slack

Official Anthropic integration. @mention in channels. Auto-routes coding tasks to Claude Code web sessions with GitHub PR creation.

Limitation: No persistent memory for non-code work. Each thread starts fresh. Requires Slack workspace.

Best for
Teams already on Slack who need code collaboration with GitHub. Less useful for marketing/content work.

‚å®Ô∏è Claude Code CLI

Most powerful option for individual development work. Full CLAUDE.md support, file system access, MCP tools.

Limitation: Single user only. Not collaborative. Terminal-based.

Best for
Individual coding work. Max's personal use. Not for team collaboration.
